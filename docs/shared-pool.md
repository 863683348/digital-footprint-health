## 一、竞品列表（PM 许清楚 写入）

### 直接竞品（删推文类）
- TweetDeleter：$4.99–7.99/月，$99.99 终身，自称 Official X.com partner；按日期/关键词/媒体筛选批量删，支持定时自动删。
- TweetDelete：$6.49–6.99/月，$99 终身；规则+自动化为主。
- Redact.dev：$7.99–14.99/月，覆盖 30+ 平台（X/Discord/Reddit 等），桌面端本地处理。
- 其他参考：Circleboom / TweetEraser / TwitWipe / Unpost / Block Party / Cyd / TweetStorm.ai。

### 替代方案（≥2）
- ① 数据经纪人移除：Optery（$3.99–24.99/月，带 before/after 截图证明）、DeleteMe（$129/年，慢）、Incogni（$7.99–14.99/月）。
- ② 数字足迹扫描器：FootprintIQ（2026 新秀，综合扫描+AI 风险评分但明确不做删除）、Have I Been Pwned、Namechk、BrandYourself。

### 竞品差评（市场空白来源）
1. 慢——X API 限速致「批量删」实际几小时到几天，与 instant 宣传不符。
2. 像诈骗——重复扣费、无退款按钮、客服回避、退款被拒。
3. 功能失效——mass deletion 不工作、连接报错、点赞/DM 删不了。
4. 流程隐瞒——需先传归档却未提前说明。
5. 普遍缺诊断——只给删除按钮，不告诉用户暴露在哪。

### 市场空白
无产品把「X 足迹扫描 + 可执行的删除」为个人用户统一。FootprintIQ 占「只诊断」位，删推文工具占「只删」位。

## 四、技术约束（架构师 高见远 写入）
- X 自 2026-02 起默认 pay-per-use，新开发者无免费读取档；Free 档仅可写（1500 帖/月），无读取权限。
- 读 $0.005/帖、写 $0.01/帖、互动 $0.015/帖；月读上限 200 万帖，超出强制 Enterprise（约 $42,000/月）。
- recent search 仅 7 天窗口；全量历史归档搜索为 Enterprise 专属 → 无法通过 API 枚举用户历史推文。
- 删除端点 DELETE /2/tweets/:id，用户级限速 50 次/15 分钟/每个授权用户；鉴权 OAuth 2.0 PKCE，scope：tweet.read + tweet.write + users.read。
- 访问令牌约 2 小时过期，需 refresh token 静默刷新。
- 推论：删除全部历史推文必须让用户下载 X 归档文件（CSV/JS），提取 tweet_id，按 50/15min 逐条 DELETE；5 万条约需 10+ 天，需可断点续传队列 + 限速退避 + 进度持久化 + 令牌刷新。

## 五、选型结论（架构师 高见远 写入）
- 前端/Web/API：Next.js（React + TypeScript），全栈一体，Vercel 部署。
- 后台常驻：独立 Node.js（TypeScript）Worker 进程（删除队列必须常驻，不可放 Serverless 短函数）。
- 数据库：PostgreSQL（含 JSONB），托管 Supabase/Neon 免费档。
- 缓存与队列：Redis（Upstash/Railway），用于 OAuth state/会话、删除限速令牌桶、BullMQ 任务队列、进度缓存。
- 对象存储：加密 S3 兼容（Cloudflare R2 / AWS S3 + SSE-KMS），存归档文件，带生命周期自动销毁。

## 六、不可行警告（架构师 高见远 写入，MVP 红线）
1. 历史推文无法靠 API 枚举（7 天窗口 + 全量搜索 Enterprise 化）→ 报告与删除必须基于用户自己下载的归档文件，不可依赖实时 API 拉历史。
2. API 读取成本高且无免费档 → MVP 严禁任何历史读取，体检报告完全由归档本地生成。
3. 删除耗时长且为付费写操作（50/15min≈4800/天；5 万条≈10+ 天；按 $0.01/写≈$500）→ 必须队列化、可暂停、可断点续传，并向用户明示费用与时长。
4. 范围边界：不做实时监听/增量删除，不做跨平台（仅 X），不做历史推文内容通过 API 回填。

## 七、设计方向（设计师 颜好看 写入）
医疗「体检报告」隐喻贯穿全站；基调=信任、平静、清晰、非恐吓。结构借 FootprintIQ/BrandYourself（总分+分项+修复计划），信任借 Optery（证据化呈现），气质借 Linear 克制 + Stripe 留白。文案平静陈述、不恐吓。含：竞品分析、Design Token 全量、5 页面 IA 与 micro-interaction、反模式自查、A11y 基础。

## 八、对标品牌（设计师 颜好看 写入）
Apple Health / 医院检验报告系统（医疗信任感）＋ Linear（清晰克制）＋ Stripe（留白信任）。竞品借鉴：TweetDeleter 筛选器、Optery 证据化暴露、FootprintIQ 仪表盘、BrandYourself 修复计划；规避其恐吓式文案与企业级冷感。

## 十、计费与预算口径（架构 3.5 补充 + 基于定价决策推导）
- 高见远已在 architecture.md 新增 **3.5 计费与预算控制模块**：实时费用累计、幂等（断点不重复计费）、`budget_cap_usd` 截断、X API 对账、前置预估卡；`delete_tasks` 表加 `budget_cap_usd / cost_incurred_usd / paused_reason`；API 加 `budgetCapUsd` 与 `costIncurredUsd / pausedReason`。
- 架构师原提 4 个待定口径，由已决「分档 + Pro」定价推导如下（产品侧已决）：
  1. **计费单位稳定性**：以「推文数 × 单价」为基准；分档 >1w = ¥0.15/帖 + ¥20 基础费；Pro 超额 = ¥0.15/帖。X API 实扣 $0.01/帖（≈¥0.07）作成本基线，售价差为毛利。
  2. **预算主维度**：双控——推文数（Pro 月度额度 2000 帖）+ 费用上限（用户可选 `budget_cap_usd`）；超时/超预算即暂停并提示。
  3. **展示币种与汇率**：面向用户展示 RMB（¥）；X 成本按 $1≈¥7.2 折算记录，汇率波动由我方吸收（毛利缓冲）。
  4. **MVP 是否接支付**：接。定价为 ToC 消费价，删除由我方 X 开发者账号代发 API、代付 X 写费，需向用户收款 → MVP 接第三方聚合支付（Stripe / 微信支付 / 支付宝），走聚合通道规避二清。*属开发阶段细节，本次规划不展开。*
