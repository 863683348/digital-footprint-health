## i18n 国际化（2026-07-13 下午）：支持中英文
- 用户指令"支持中英文"。经 AskUserQuestion 确认两点：①语言策略=自动识别浏览器 locale + 手动切换(记忆 localStorage)；②本地化范围=仅前端界面（后端返回结构化 key，前端翻译）。
- 实现：新增 lib/i18n.ts（zh/en 双目录 + translate/translateError，支持 {var} 插值、错误码映射+fallback）+ components/I18nProvider.tsx（Context + useI18n() + useLang，'use client'）。app/layout.tsx 用 I18nProvider 包裹全局，并据语言设置 document.documentElement.lang。
- 后端配合改为结构化 key：scoring.ts 风险旗去掉中文 label，仅留 kind+severity；payment.ts BILLING_PLANS 去掉 name/desc，BillingInfo 改 refundPolicyKey（前端按 plan.<id>.name/desc 与 refund.policy 翻译）。types.ts 同步（RiskFlag.label?、BillingPlan/BillingInfo 精简）。
- 前端全部文案走 t()：NavBar 加「中/EN」切换器；5 页面 + Footer + ui.tsx(FeeEstimateCard) + ScoreGauge + Loading 组件接 useI18n；维度标签/风险旗/档位/状态/错误码均按 key 翻译。api-client.ts 抛 ApiError(code)，页面用 te(code,msg) 本地化错误。
- 验证：tsc 0 报错；3100 页面全 200；headless 校验 EN 目录 9/10（唯一"FAIL"是测试预期误写千分位，实际英文串正确）；SSR 默认 zh（HTML 含「中/EN」+中文 hero），EN 浏览器访问或点 EN 后切换英文。
