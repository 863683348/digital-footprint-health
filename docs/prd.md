## 0. 一句话定位

**「数字足迹体检报告」是一款面向隐私焦虑个人用户的 Web 工具：先把你 X 账号的暴露面「看清楚、打个分」，再给你一剂「近似一键清空历史推文」的可执行处方——把「诊断」和「治疗」在一个干净闭环里做完。所有数据来自你自己下载的 X 归档文件，报告零 API 读成本，删除是透明计费的付费写操作。**

---

## 0.5 硬性技术约束（来自架构师高见远，PRD 必须遵守）

> 以下为架构层不可违反的约束，PRD 全文须与之对齐；详细不可行警告已单列进共享内存池，可直接引用。

- **C1 数据源只能来自用户自下载的 X 归档文件（CSV/JS）**：禁止在需求中写「通过 API 自动拉取历史推文」。原因——X API 仅 7 天窗口、全量历史搜索是 Enterprise 档（约 $42k/月），根本枚举不到历史推文。**MVP 唯一入口 = 用户下载官方归档 zip → 上传。**
- **C2 MVP 严禁任何 X API 读取**：读操作 $0.005/帖且无免费档。体检报告须完全**本地由归档生成**，零 API 读成本、零读延迟风险。
- **C3 删除是付费写操作（非免费、非 instant）**：写操作 $0.01/帖；5 万帖 ≈ $500；限速 50 帖/15 分钟 ⇒ 5 万帖 ≈ 10+ 天。需求须体现：**队列化、可暂停、可断点续传、费用/时长预期前置展示**。
- **C4 推荐分阶段交付**：Stage 1 = 体检报告（零读成本、最快验证、最低风险）；Stage 2 = 删除（付费写，需成本/时长预期与计费）。

**对 PRD 的影响**：删除不再是「免费一键魔法」，而是「透明计费、慢、可管理」的治疗手段。我们的差异化从「更快删除」转向「诚实预期 + 可见进度 + 安全护栏」——正面回应竞品「慢/像诈骗」差评。

## 1. 核心问题陈述（真实需求，而非表面方案）

### 1.1 用户嘴上说的 vs 真实需要的
- 用户嘴上说：「帮我删掉 X 上的旧推文」「给我个群打卡/清理工具」。
- 真实需要：**对自己的 X 数字足迹「既看不清、也不敢删、删不净」，需要一个可信赖的闭环来同时解决「看清暴露」和「安全地清理掉」两件事。**

### 1.2 问题陈述（Problem Statement）
| 维度 | 描述 |
|------|------|
| **谁** | 隐私焦虑的个人用户（求职/转型期职场人、数字极简主义者、曾被旧言论困扰过的人） |
| **场景** | 面试前、换工作、被人肉搜索后、定期做「数据 detox」时，想快速评估并清理自己在 X 上的历史暴露 |
| **痛点** | ① 看不清：手动翻旧推文不现实，不知道自己到底暴露了什么；② 不敢删：怕误删重要内容、怕删不干净、怕被乱扣费；③ 删不净/不值：现有删推文工具贵、慢、像诈骗（见竞品差评）；扫描器只查不给方案 |
| **现有解法** | 手动删（耗时）、付费删推文工具（贵/慢/信任差）、隐私扫描器（只诊断不治疗）、数据经纪人移除（不碰社交平台） |
| **为什么不行** | 工具严重割裂——要么只删不诊断，要么只诊断不删。用户被迫在「昂贵的清理工具」和「不给方案的扫描器」之间二选一，没有「先体检、再开药」的完整体验 |

### 1.3 我们的切口（市场空白）
- 绝大多数工具只覆盖**单一维度**：删推文 **OR** 删数据经纪人 **OR** 仅扫描。
- FootprintIQ（2026 新秀）做综合扫描 + 风险评分，但**明确不做自动删除**。
- **目前没有产品把「X 足迹扫描 + 可执行的删除」为个人用户干净地统一起来。**
- 我们的切口 = **聚焦 X 一个数据源，给出清晰健康分 + 近似一键的历史推文删除**，形成「诊断→治疗」闭环。

## 2. 目标用户画像

### 2.1 主画像：求职焦虑的「小林」（28 岁，职场转型期）
- **身份**：一线城市互联网/外企职场人，正在或准备求职/跳槽；部分有海外背景、X 使用频繁。
- **技术水平**：中等。会用 X，但不知道「数据归档下载」「API 限速/计费」等概念——**产品必须把「下载归档」做成图文引导，不能假设用户懂技术**。
- **核心焦虑**：担心年轻时/情绪化/争议性的旧推文被未来雇主、同事、客户搜到，影响机会。
- **典型场景**：
  - 收到面试通知当晚，打开 X 想自查，翻了 20 分钟就放弃（太累、太多）。
  - 想清理但怕误删现在仍有价值的推文，也怕「删了但没删净」——毕竟 X 搜索/缓存/第三方可能还留着。
  - 试过某删推文 App，付了钱删不动/删很慢，客服不理，退款被拒，从此不信任这类工具。
- **付费意愿**：中高。愿意为「省心 + 可信 + 看得见的清理结果」付费，但极度反感「付一次款跑路」「退款难」「偷偷扣费」。对「删除要花时间和钱」可接受，前提是**提前说清楚**。

### 2.2 次画像：数字极简主义者「老陈」（35 岁，定期数据 detox）
- **身份**：关注数字隐私的资深用户，把清理 X 当作「数字卫生」的一部分。
- **核心诉求**：周期性体检 + 定时自动清理（保留最近 N 天后自动删），维持低暴露基线。
- **付费意愿**：中。更看重自动化和可持续性，倾向订阅。

### 2.3 非目标用户（暂不在 MVP 范围）
- 企业/HR/背调机构（B 端批量筛查）——本产品是「个人替自己清理」，立场相反。
- 需要多平台（Discord/Reddit/FB 等）统一治理的用户——MVP 仅 X。

## 3. 竞品分析

### 3.1 直接竞品（删推文类，≥3）
| 竞品 | 核心功能 | 优势 | 劣势（来自差评/吐槽） | 定价 |
|------|----------|------|----------------------|------|
| **TweetDeleter** | OAuth 连接 + 上传 X 归档 → 按日期/关键词/媒体批量删；自称 Official X partner | 品牌可信、筛选维度多、支持定时自动删 | 「Unlimited 只一次删 100 条、要几小时到几天」；点赞删除失效；**退款被拒、客服回避**；需先传归档但未提前说明 | $4.99–7.99/月，$99.99 终身 |
| **TweetDelete** | 规则 + 自动化删除为主 | 自动任务「设一次持续清」体验好 | **被大量投诉自动续费/重复扣款、退款无门、质疑诈骗**；客服差；「产品根本用不了、两个月邮件循环」 | $6.49–6.99/月，$99 终身 |
| **Redact.dev** | 覆盖 30+ 平台（X/Discord/Reddit…），桌面端**本地处理** | 本地处理隐私友好、平台广 | 平台多但单平台不深；桌面端门槛高；删点赞/DM 受平台限速更慢 | $7.99–14.99/月 |
| *其他参考* | Circleboom、TweetEraser、TwitWipe、Unpost、Block Party、Cyd、TweetStorm.ai | — | App 端「mass deletion never works」「charges you but does nothing」差评集中 | 各异 |

### 3.2 替代/相邻方案（≥2）
| 方案 | 类型 | 能做什么 | 与我们的关系 |
|------|------|----------|--------------|
| **Optery / DeleteMe / Incogni** | 数据经纪人移除 | 帮用户从数据经纪人处 opt-out；Optery 带 before/after 截图证明 | **替代但不重叠**：解决「经纪人数据」而非「X 社交足迹」；且 DeleteMe 慢（3 个月仅移除 27%）、Incogni 无法验证是否真有你的数据 |
| **FootprintIQ / Have I Been Pwned / Namechk / BrandYourself** | 数字足迹扫描器 | 综合扫描（用户名/泄露/经纪人/暗网）+ AI 风险评分；HIBP 查泄露、Namechk 跨 90–100 平台查用户名 | **关键替代/警示**：FootprintIQ 做「综合扫描 + 评分」但**明确不做自动删除**——即「只诊断不治疗」。这是我们的直接差异化靶子 |

### 3.3 竞品差评提炼（最有价值的情报）
1. **慢**：X API 限速导致「批量删」实际是几小时到几天，与「instant」宣传严重不符。
2. **不可信/像诈骗**：重复扣费、无退款按钮、客服装傻、退款被拒，信任崩塌。
3. **功能失效**：mass deletion 不工作、连接报错、「点赞/DM 删不了」（受平台限制）。
4. **流程隐瞒**：要求先传归档文件但未在购买前说明，引发「货不对板」感。
5. **缺诊断**：所有删推文工具都不告诉用户「你到底暴露在哪、风险多高」——只给一个删除按钮。

### 3.4 我们的差异化（回答「用户为什么选我们」）
- **闭环**：别人「只删不诊断」或「只诊断不删」；我们「先体检评分、再一键清理」。
- **先免费看清，再付费治疗**：用免费体检报告建立信任，规避「付了款才发现不好用」的信任陷阱。
- **诚实的删除预期（核心差异化）**：删除前**明确展示费用 + 时长估算**，进度可见、可暂停、可断点续传——正面回应竞品「慢/像诈骗/隐瞒」三大差评，绝不承诺虚假 instant。
- **安全护栏**：删除前二次确认、可预览待删清单、可设保护列表，解决「不敢删」。
- **聚焦 X 做深**：不贪多平台，把单数据源的体验打磨到竞品做不到的顺滑。
- **零读成本的诊断**：报告完全本地由归档生成，不受 X API 读计费/限速影响（C2）。
- **退款透明、无门槛（核心信任闭环）**：正面回应竞品「退款难/像诈骗」痛点——提供清晰退款入口与规则（未开始全额退、进行中按已完成比例退），把「敢付费」做成信任闭环的最后一块（详见 §9.5）。

## 4. MVP 功能清单（P0 / P1，含交付分期）

> MVP 只做 **X 单数据源（归档-only）+ 体检报告 + 近似一键删除** 的闭环。P0 = 核心价值闭环必须项；P1 = 显著增强、紧随其后。**交付分期**：Stage 1（零读成本、最快验证）先行，Stage 2（付费写）紧随。

### P0 · Stage 1（首交付，零 API 读成本，最快验证）
| 编号 | 功能 | 验收标准摘要 |
|------|------|--------------|
| F1 | **X 归档文件上传与解析**（C1 唯一入口） | 用户下载 X 官方归档 zip → 上传 → 系统本地解析 tweets.js/账户信息（**不调任何 X 读取 API**），正确提取推文总数、时间跨度、媒体、@对象等元信息；解析失败有清晰报错与重试指引；提供「如何下载归档」图文引导 |
| F2 | **体检报告生成 + 风险评分**（C2 本地生成） | 完全**本地由归档计算**「健康分 0–100」与风险等级（低/中/高），覆盖维度：推文量级、含 PII（邮箱/电话/地址）、含定位、敏感/争议语义、含媒体、旧推文占比、账号信息暴露；输出分项风险与示例高危推文；零 API 读成本 |
| F3 | **风险可视化仪表盘** | Web 页展示总分、分项雷达/条形、Top N 高危推文样例、关键暴露指标，5 秒内可读懂自身暴露面 |

### P0 · Stage 2（付费写操作，C3 约束）
| 编号 | 功能 | 验收标准摘要 |
|------|------|--------------|
| F4 | **近似一键删除历史推文（含安全护栏 + 队列 + 成本/时长预期）** | 删除前**必须先展示**：待删条数、预计 X API 写费用（≈$0.01/帖）、预计耗时（限速 50/15min）；用户确认付费后进入**队列**，分批调用 X 删除 API，进度实时可见、可暂停/取消、**断点续传**；可设保护列表（收藏/指定推文）；完成后给「删除完成报告（已删/失败/受保护）」 |

### P1（MVP 紧随）
| 编号 | 功能 | 验收标准摘要 |
|------|------|--------------|
| F5 | **选择性删除筛选** | 支持按日期区间 / 关键词 / 是否含媒体 筛选删除，不止「全删」（仍受 C3 计费/限速约束，筛选后同样前置展示费用与时长） |
| F6 | **报告 PDF 导出** | 一键导出体检报告 PDF，供留存/分享（本地生成，无 API 读） |
| F7 | **删除后复检** | 删除完成后可重新体检，量化「暴露下降了多少」 |
| F8 | **定时自动清理** | 设置「保留最近 N 天，超期自动删」，持续维持低暴露基线（服务次画像老陈；仍受 C3 计费/限速，须展示累计费用预期） |

> P2（后续迭代，非本次）：F9 OAuth 连接（仅用于删除写操作的授权/便捷重连，**仍无法绕过归档**读取历史，因 C1 限制历史只能来自归档）、多平台扩展、点赞/DM 删除、AI 改写保留等。

## 5. RICE 评分表（Reach / Impact / Confidence / Effort）

> 评分说明：Reach 取目标用户触达相对值（1–10）；Impact：0.25 极小 / 0.5 低 / 1 中 / 2 高 / 3 极大；Confidence：0.5 低 / 0.8 中 / 1.0 高；Effort 单位=人周。`RICE = (Reach × Impact × Confidence) / Effort`。

| 编号 | 功能 | 优先级 | 分期 | Reach | Impact | Confidence | Effort(人周) | **RICE** | MVP? |
|------|------|--------|------|-------|--------|------------|--------------|----------|------|
| F1 | X 归档上传与解析 | P0 | S1 | 10 | 3 | 1.0 | 2 | **15.00** | ✅ |
| F3 | 风险可视化仪表盘 | P0 | S1 | 10 | 2 | 0.9 | 2 | **9.00** | ✅ |
| F2 | 体检报告 + 风险评分 | P0 | S1 | 10 | 3 | 0.9 | 4 | **6.75** | ✅ |
| F6 | 报告 PDF 导出 | P1 | S1+ | 5 | 1 | 0.9 | 1 | **4.50** | ⬜ |
| F4 | 近似一键删除 + 护栏 + 计费预期 | P0 | S2 | 8 | 3 | 0.8 | 6 | **3.20** | ✅ |
| F7 | 删除后复检 | P1 | S2+ | 6 | 1.5 | 0.8 | 2 | **3.60** | ⬜ |
| F5 | 选择性删除筛选 | P1 | S2+ | 6 | 2 | 0.8 | 3 | **3.20** | ⬜ |
| F8 | 定时自动清理 | P1 | S2+ | 4 | 2 | 0.7 | 4 | **1.40** | ⬜ |
| F9 | OAuth 连接（仅写授权） | P2 | — | 7 | 1.5 | 0.6 | 5 | **1.26** | ⬜ |

**排序解读与战略覆写说明**：
- RICE 按 ROI 排序为 F1 > F3 > F2 > F6 > F4 ≈ F5 > F7 > F8 > F9。
- **交付分期独立于 RICE**：F1/F2/F3（Stage 1，零读成本）先行交付，以最低风险最快验证「诊断」价值与转化漏斗；F4（Stage 2，付费写）紧随。此分期直接采用架构师建议（C4）。
- **F4 强制 P0**：虽 RICE 居中（Effort 因队列/暂停/续传/计费抬高至 6），却是差异化「治疗」唯一承载——若不交付，我们沦为又一个「只诊断不删」的扫描器（FootprintIQ 已占位）。
- F6（PDF 导出）RICE 偏高但属「锦上添花」，列 P1 不影响闭环。

## 6. MVP 范围边界（明确不做 / Won't Have）

| 不做项 | 原因 |
|--------|------|
| **通过 X API 自动拉取历史推文** | **技术不可行（C1）**：API 仅 7 天窗口、全量搜索 Enterprise $42k/月，无法枚举历史。MVP 唯一入口 = 用户自下载归档 |
| **MVP 任何 X API 读取** | **约束（C2）**：读 $0.005/帖且无免费档；报告须本地由归档生成，零读成本 |
| **多平台覆盖**（Discord/Reddit/FB/IG 等） | 聚焦 X 做深；Redact 覆盖 30+ 但单平台不深，我们要避免 scope creep |
| **数据经纪人 opt-out**（DeleteMe/Incogni 类） | 业务模型不同，需人工/合规流程，远超 MVP |
| **泄露库 / 暗网扫描**（HIBP 类） | 需外部数据源与授权，且非 X 范畴 |
| **点赞 / 转发 / DM 删除** | X API 限速导致慢且易失败（竞品差评重灾区）；MVP 只删 tweets，规避信任陷阱 |
| **OAuth 实时连接作为历史读取入口** | 受 C1 限制，OAuth 即使接入也读不到历史（仅 7 天），故 MVP 不做；P2 仅作删除写授权辅助 |
| **AI 改写/美化旧推文（保留但改写）** | 超出「删除治疗」范畴，属另一个产品方向 |
| **企业版 / 团队协作 / 背调** | 立场相反（个人替自己清理），C 端优先 |
| **常驻授权 / 长期托管账号** | 隐私信任风险高，MVP 不做账号常驻 |
| **多语言（i18n）** | MVP 中文界面优先，验证国内隐私焦虑用户需求 |

## 7. 验收标准摘要（Given / When / Then）

> 先列要点；详细 AC 在研发阶段由架构师/测试补全。

**F1 归档上传解析（C1）**
- Given 用户已下载 X 官方归档 zip，When 上传并解析，Then 系统**本地**提取推文总数/时间跨度/媒体数/@对象，**全程不调用 X 读取 API**，展示解析摘要；解析失败给出明确错误与重试指引。
- Given 用户不知如何下载归档，When 进入上传页，Then 展示图文引导（设置→下载归档）。

**F2 体检报告 + 风险评分（C2）**
- Given 解析完成，When 触发体检，Then **本地**生成 0–100 健康分 + 风险等级 + 分项风险（PII/定位/敏感语义/媒体/旧推文占比/账号暴露）+ ≥3 条示例高危推文；零 API 读成本。
- Given 归档为空或极少量推文，When 体检，Then 给出「样本不足」提示而非错误分。

**F3 风险可视化仪表盘**
- Given 报告已生成，When 用户打开仪表盘，Then 5 秒内可读懂总分、分项图形、Top 高危样例。

**F4 近似一键删除 + 护栏 + 成本/时长预期（C3）**
- Given 用户查看待删清单并设好保护列表，When 点「确认删除」，Then **先弹出费用/时长预估卡**：待删 N 条、预计 X API 写费用 ≈$X（¥Y）、预计耗时 ≈Z 天（限速 50/15min）；用户二次确认付费后生成删除**队列**。
- Given 删除队列运行中，When 用户点暂停，When 再点恢复，Then 从断点续删，不重复删已删项；进度条实时更新。
- Given 删除进行中遭遇 X API 429/限速，When 重试，Then 自动按限速退避重试，状态可见。
- Given 删除完成，When 查看，Then 展示「已删 X 条 / 失败 Y 条 / 受保护 Z 条」完成报告与实付费用。
- Given 待删清单包含保护列表内推文，When 执行删除，Then 该部分被跳过并标注。

**F6 报告 PDF 导出（P1）**
- Given 报告已生成，When 点导出，Then 下载结构化 PDF（含总分、分项、高危样例），本地生成无 API 读。

**F7 删除后复检（P1）**
- Given 已完成一次删除，When 重新体检，Then 报告体现暴露指标下降并量化「下降 X%」。

## 8. 边界条件（非功能 / 异常态）

- **空状态**：未上传归档 → 引导下载 X 归档的图文步骤。
- **错误状态**：归档格式错误/损坏 → 明确报错 + 重新下载指引；X API 删除失败（401/429）→ 重试 + 限速提示，**不丢已删进度**（C3 断点续传）。
- **加载状态**：大归档（数万条）解析显示进度（本地、无 API 等待）；删除任务长跑显示预计剩余时间与实时费用累计。
- **边界值**：单条超长推文、含 emoji/特殊字符、零推文账户、归档极老（字段变更）需兼容。
- **并发**：同一账号不允许两个删除任务并行（防重复删、防重复计费）。
- **离线**：删除进行中断网 → 本地记录进度，恢复后续删；不丢状态、不重复计费。
- **权限拒绝**：X 写授权失败/用户撤销 → 删除任务挂起并提示重新授权，已删部分保留。
- **隐私/合规**：归档文件解析后不长期存储原文；明确告知数据存储与删除策略，建立信任（直面竞品「像诈骗」痛点）。
- **计费透明（C3 衍生）**：任何删除操作前必须展示费用与时长估算；实际扣费以 X API 实扣为准，差额向用户说明。

## 9. 商业模式（已决 · 用户 2026-07-13 确认）

> 参考竞品定价：TweetDeleter $4.99–7.99/月（$99.99 终身）、TweetDelete $6.49–6.99/月（$99 终身）、Redact $7.99–14.99/月、DeleteMe $129/年、Optery $3.99–24.99/月（含免费档）、Incogni $7.99–14.99/月。
> **成本覆盖逻辑（来自 C3）**：删除是 X API 付费写操作，基准写费 ≈ $0.01/帖（≈¥0.07/帖）。本模型所有档位价均已内含该 API 写成本 + 我方毛利，故不再亏损。下单前即向前置展示「费用 + 时长预估」，正面回应竞品「慢/像诈骗/隐瞒」三大差评。

**核心思路：Freemium「先免费看清，再按需付费治疗」，刻意避开「终身制 + 退款难」的信任陷阱；删除费用透明、按量/分档计费。定价模型 = 单次清理分档 + Pro 订阅组合（已决）。**

### 9.1 定价模型（已决）

| 档位 | 价格 | 包含 | 成本覆盖说明 |
|------|------|------|--------------|
| **Free（引流）** | ¥0 | 上传归档 → 体检报告 + 健康分 + 风险清单（示例 Top N） | 零成本获客（C2 本地生成无 API 读费）；建立信任、证明价值 |
| **单次清理 · 分档** | ≤1,000 帖：**¥29/次**<br>1,000–10,000 帖：**¥99/次**<br>>10,000 帖：**¥0.15/帖 + 基础服务费 ¥20/次** | 一键删除全部历史推文（含费用/时长预估 + 队列 + 安全护栏 + 完成报告） | 各档价已内含 X API 写费（¥0.07/帖基准）+ 毛利；>1万档显式覆盖成本且留利，不再亏损 |
| **Pro 订阅** | **¥29–49/月 或 ¥299/年** | 无限次体检 + 选择性删除 + 定时自动清理 + PDF 报告 + 复检；含**月度删除额度 2,000 帖/月**，额度内包 X API 写费，超额按 ¥0.15/帖计 | 额度内 API 费由订阅费覆盖；超额按量，沿用分档单价 |

### 9.2 有意不做的定价
- **不卖终身档**：竞品终身档（$99）常被吐槽「退款难、付完跑路」，与我们要建立的信任相悖。
- **不做「固定低价包删全部」**：受 C3 成本约束，固定低价在大量推文下必亏——已由分档（>1万按量）规避。

### 9.3 待验证假设（MVP 上线后 A/B 与漏斗验证）
1. 免费体检报告能否有效转化为删除付费（诊断→治疗漏斗）？
2. **费用/时长透明前置，是否真能驱动付费（尤其大量推文用户是否愿为「透明慢删」买单）？档位接受度如何（¥29 / ¥99 / 按量）？**
3. 「健康分 + 高危样例」的焦虑触达，是否足以驱动首次付费？
4. 用户对「删除需 10+ 天、按条计费」的接受度（管理预期是否成功）？
5. 中文隐私焦虑用户对「X 单数据源」的付费意愿基线？Pro 订阅 vs 单次分档，哪种 LTV 更优？

### 9.4 计费口径（全部已决 · 用户 2026-07-13 确认，对齐架构 §3.5）
以下 4 个口径**全部已决（用户 2026-07-13 确认）**，供架构计费模块（docs/architecture.md §3.5）实现，已与高见远确认：
1. **计费单位（稳定假设）**：成功删除 1 条 X 内容 = 1 次 write = $0.01。`cost_incurred_usd` 累加口径 = 仅以「X DELETE 成功返回（与 `succeeded` 同事务）的条目数」为准；失败不计、预删不计。MVP 删除范围仅 tweets：含用户原创 tweet 与 quote tweet（均属用户发出的 tweet，删 = 1 write = $0.01）；**不含** likes / DM（Won't Have）；retweet 不做专项清理（若随「全删」包含，每条成功删除同样计 $0.01）。即 MVP 删除流程中**无其它被 X 计为写的操作混入**。
2. **预算上限主维度 = 金额（USD 基准，可显本地币种）**：预估卡与预算输入以「本次最多花 $X」为主；条数作为关联展示（「$X 约可删 N 条」）。理由：直接对应用户「不想被乱扣费」核心焦虑，且与分档定价（按条计费）正交——分档定单价，金额上限定本次天花板。
3. **展示币种与汇率**：基准 USD（对齐 X 账单）；MVP 叠加本地币种 **¥** 展示。汇率口径 = **固定 1 USD = 7.2 CNY**（写死配置，零外部依赖），页面标注「汇率按固定 1:7.2 估算，仅供参考」。后续接支付后再升级为每日更新。
4. **MVP 计费模式 = 代缴 + 接支付（否定纯透传）**：v1.0 定价已决且明确「档位价内含 X API 写费 + 毛利」，故平台代收用户费用（¥）、平台代缴 X 写费（$0.01/条，用平台 API 额度），平台留毛利。纯透传（用户自担、我们不收费不代缴）与已决定价矛盾，不采用。MVP 直接接支付（分档 + Pro 全上线），支付渠道 / 退款 / KYC 详见 §9.5（已决）。

### 9.5 支付与退款政策（已决 · 用户 2026-07-13 确认）
MVP **直接接支付**，分档（≤1k / 1k–1w / >1w）+ Pro 订阅（¥29–49/月 或 ¥299/年，含 2,000 帖/月额度）全量上线，实现完整「诊断→治疗」闭环（Q4 已决）。
- **支付渠道**：接入第三方支付；vendor 建议 Stripe / 国内聚合支付，具体选型开发阶段定。平台代收用户（¥）、平台代缴 X 写费（$0.01/条），留存毛利。
- **退款政策（正面回应竞品「退款难 / 像诈骗」痛点）**：提供清晰、无门槛的退款入口与规则——删除**未开始**全额退；删除**进行中**按已完成比例退（剩余未删部分退款）；完成后不支持退（已交付）。规则在产品内显式展示，把「敢付费」做成信任闭环。
- **Pro 订阅额度托管**：2,000 帖/月额度核销；超额按 ¥0.15/帖 计（与分档 >1w 单价一致）。
- **KYC / 税务边界**：列为开发阶段待定，本次规划不展开。
