## 二、核心结论（一句话）

面向隐私焦虑 C 端个人，做一款 **X 单数据源**的「数字足迹体检报告」Web 工具：先免费把暴露面**看清楚、打个分**，再提供**近似一键删除历史推文**的清理手段，形成「诊断→治疗」闭环，填补 FootprintIQ（只诊断不删）与删推文工具（只删不诊断）之间的空白。

## 三、关键决策（Decision Log）

- `[Phase 0]` 锁定四要素：推特定位=体检+删推文两者都要；用户=C 端隐私焦虑个人；数据范围=仅 X；形态=Web 应用。
- `[Phase 1]` MVP 必含「删除」(F4, P0) —— 用户明确"两者都要"，故覆盖 FootprintIQ 只诊断不删的空白，不采纳架构师"删除后置"的折中建议。
- `[Phase 1]` 采用**归档驱动删除**（非 API 枚举）—— X API 仅 7 天搜索窗口、全量历史搜索需 Enterprise（$42k/月）、无免费读档；报告与删除均基于用户自下载的 X 归档文件。
- `[Phase 1]` 设计对标 = Apple Health 医疗信任感 + Linear 克制 + Stripe 留白；浅色主题；主色冷静临床蓝 #2E5BCB；语义三态绿/琥珀/红仅作风险指示。
- `[Phase 1]` 技术栈 = Next.js 全栈 + 独立 Node Worker（删除队列常驻）+ PostgreSQL + Redis(BullMQ) + 加密 S3 归档存储。
- `[Phase 1]` 删除定价定为「分档 + Pro 组合」（已决，用户 2026-07-13 确认）：Free ¥0 / 单次 ≤1k ¥29、1k–1w ¥99、>1w ¥0.15/帖+¥20 基础费 / Pro ¥29–49 月或 ¥299 年含 2k 帖月额度；成本覆盖 X API 写费 ¥0.07/帖基准，无亏损风险。
- `[Phase 1]` PRD 经 v0.2（对齐架构 C1–C4 硬约束）→ v1.0（定价定稿·已确认），与架构/UIUX 现已零冲突。
- `[Phase 1]` 支付决策 = MVP 直接接支付（分档+Pro 全上线）；PRD §9.5 封版为已决，含无门槛退款（未开始全额退 / 进行中按比例退），正面回应竞品「退款难」痛点；vendor / KYC 开发阶段定。

## 五、需用户知晓的现实约束（删除成本）

- X API 删除限速 **50 次 / 15 分钟** ≈ 4800 条/天；5 万条历史推文 ≈ **10+ 天**完成。
- 每次 DELETE 计为写操作，pay-per-use **$0.01/条** → 5 万条 ≈ **$500**（或耗尽 Free 档 1500 写/月额度）。
- MVP 化解方式：归档驱动 + 队列化 + 可暂停/断点续传 + 每日上限/日期范围 + 费用与时长**明示**，不承诺虚假的"instant"。
- 架构师个人建议"先上体检报告单能力、删除作第二阶段"，但你的"两者都要"已覆盖该决策，故按两者都做推进，护栏写入设计。
